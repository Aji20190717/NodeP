JAVA 
JAVA는 네트워크상에서 쓸수 있도록 미국의 선마이크로시스템즈가 개발한 객체지향프로그래밍언어 

JAVA의특징 
자바가상머신(JVM)만 설치하면 컴퓨터의 운영체제에 상관없이 작동한다. (즉, 운영체제에 독립적) 
기본자료형을 제외한 모든요소들이 객체로 표현 
객체지향개념의 특징인 캡슐화, 상속, 다형성이 잘 적용된 언어 
Garbage Collector를 통한 자동적인 메모리관리
멀티쓰레드(Multi-thread)를 지원 

Garbage Collection(가비지컬렉션) 
시스템에서 더이상 사용하지 않는 동적 할당된 메모리 블럭을 찾아 자동으로 다시 사용 가능한 자원으로 회수하는 것으로 시스템에서 가비지컬렉션을 수행하는 부분을 가비지컬렉터라 부른다. 

OOP (객체지향프로그래밍) 
OOP란 Object-Oriented Programming의 약어로써 객체지향 프로그래밍을 의미 
데이터를 객체로 취급하여 프로그램에 반영한 것이며, 순차적으로 프로그램이 동작하는 기존의 것들과는 다르게 객체와 객체의 상호작용을통해 프로그램이 동작하는것을 말한다. 

- 참고 
<절차지향 vs 객체지향> 
1.  절차지향(명령형): 컴퓨터 작업처리 방식과 유사하여 순차적으로 진행되기 때문에 흐름파악이 쉽고 시간적으로 유리 
하지만 점차 하드웨어가 소프트웨어의 발달을 따라오지 못하며 객체지향 언어가 등장. 
객체지향의 기능별 모듈화를 통해 중복을 최소화하고 모듈 재사용하여 하드웨어 처리량을 획기적으로 감소 
(C언어, basic, fortran, algol 등) 

2.  객체지향: 객체간의 관계/조직에 초점을두고 실제 세계를 모델링하여 소프트웨어를 개발하는 방법 
특징: 캡슐화(정보은닉), 상속, 다형성, 추상화 
(JAVA, C++, C#, delphi 등) 

3. 스크립트언어: 컴파일없이 실행되며 최종사용자가 사용자 요구에 맞게 응요 프로그램 동작을 수행 가능 
(Python, Javascript, PHP, Perl 등) 

4. 선언형 언어: 해법 정의보다는 문제를 설명하는 언어로, ‘무엇’을 한것인지에 중점 
• 함수형 언어: 순수함수, 익명함수(람다식), 고계함수 
• 논리형 언어: 사실, 규칙, 질문 
• 특수 분야 언어: SQL
